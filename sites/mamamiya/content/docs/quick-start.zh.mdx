title: 快速开始
description: 用最少配置把 Roller Rabbit 跑起来（本地开发、数据库、管理员权限、上线）。

在开始之前，请确保你已准备好：

- Node.js 20+
- pnpm
- PostgreSQL（如果你需要登录、管理后台、RBAC、设置或支付等功能）

## 本地开发

### 初始化项目

如果你已经在仓库目录中，可以从「安装依赖」开始。

1. 获取代码

```bash
git clone "https://github.com/Larrybin/aooi.git" "my-project"
cd "my-project"
```

2. 安装依赖

```bash
pnpm install
```

3. 启动开发服务器

```bash
pnpm dev:local
```

默认端口为 `3000`：`http://localhost:3000`。自定义端口示例：

```bash
pnpm dev:local --port 8080
```

### 配置环境变量

1. 创建本地开发环境变量文件

```bash
cp ".env.example" ".env.development"
```

2. 修改 `.env.development`

本地配置的实际规则：

- `pnpm dev:local`：选择 `sites/dev-local/site.config.json` 作为本地开发站点。
- 本地站点 origin 是 `http://localhost:3000`。
- 如需本地运行其他站点，使用 `SITE=<site-key> pnpm dev`。
- 生产态、Cloudflare、smoke、build、deploy 命令必须显式设置目标站点，例如 `SITE=mamamiya`。

如果你需要登录、管理后台、RBAC、设置或支付等数据库功能，必须配置：

- `DATABASE_URL`：PostgreSQL 连接串
- `DATABASE_PROVIDER=postgresql`
- `DB_SINGLETON_ENABLED=true`
- `BETTER_AUTH_SECRET` 和 `AUTH_SECRET`：共享鉴权密钥。`pnpm dev:local` 会在两者都缺失时注入本地 fallback secret，但显式设置更利于本地脚本和应用保持一致。

示例（按需替换为你的值）：

```bash
# database
DATABASE_URL="postgresql://user:password@127.0.0.1:5432/my_app"
DATABASE_PROVIDER="postgresql"
DB_SINGLETON_ENABLED="true"

# auth secret
# openssl rand -base64 32
BETTER_AUTH_SECRET="your-secret"
AUTH_SECRET="your-secret"
```

生成随机密钥：

```bash
openssl rand -base64 32
```

说明：

- `DATABASE_PROVIDER` 当前仅支持 `postgresql`。
- `DB_SINGLETON_ENABLED=true` 只作用于本地和其他非 Workers Node 运行时。Cloudflare Workers 运行时使用 `HYPERDRIVE` binding。
- `BETTER_AUTH_SECRET` 优先级高于 `AUTH_SECRET`；除非有明确原因，建议两者使用同一个值。
- 站点 identity 来自 `sites/<site-key>/site.config.json`。生产环境如果设置 `NEXT_PUBLIC_APP_URL`，需要与 `site.brand.appUrl` 保持一致。

### 配置数据库

如果你的项目需要登录、管理后台、RBAC、设置或支付等功能，必须先配置数据库。

1. 创建 PostgreSQL 数据库，拿到连接串后写入 `DATABASE_URL`。

2. 迁移数据表

```bash
pnpm db:migrate
```

可选：做一次 schema 自检（对齐迁移与表结构）

```bash
pnpm db:check
```

如果迁移命令卡住或超时，优先用 `psql` 验证连通性：

```bash
psql "postgresql://user:password@host:port/database"
```

### 配置管理后台访问

管理后台路径为 `/admin`（也支持 `/<locale>/admin`）。首次访问会被重定向到 `/sign-in`。

1. 初始化 RBAC（写入默认角色与权限）

```bash
pnpm rbac:init
```

2. 在网页注册一个账号（邮箱 + 密码），然后为该账号分配 `super_admin`：

```bash
pnpm rbac:assign -- --email="admin@your-domain.com" --role="super_admin"
```

3. 再次访问 `/admin`，即可进入管理后台。

### 品牌配置（推荐）

构建期的站点身份默认值保存在 `sites/mamamiya/site.config.json`。登录管理后台后，可通过 `Admin -> Settings -> General -> Brand` 在运行时查看或更新当前站点的品牌信息：

- `App Name`：用于站点标题、文档/SEO、邮件标题等
- `App URL (Origin)`：用于 canonical / sitemap / callback 等 URL 生成（必须是纯 origin）
- `Support Email`：用于法律页面与联系入口
- `App Logo` / `Favicon` / `Preview Image`：可直接在后台上传（需先配置 `Storage`）

注意：`sites/<site-key>/site.config.json` 是站点 identity 来源。生产环境如果设置 `NEXT_PUBLIC_APP_URL`，需要与 `App URL (Origin)` 保持一致。

## 功能配置

项目发布上线后，通过 `/admin/settings/*` 进入管理后台按需开启功能开关并配置运营值。

判断一个值应该放在哪里时，按这个边界处理：

- Runtime env、secrets、Cloudflare bindings 负责应用接线和凭据。相关 Admin Settings 生效前，先配置这些值：`DATABASE_URL` 或 Hyperdrive、`BETTER_AUTH_SECRET` / `AUTH_SECRET`、OAuth client ID/secret、支付 provider API key/webhook secret、R2 bucket bindings、`STORAGE_PUBLIC_BASE_URL`、Images 和 Workers AI bindings。
- Admin Settings 保存非 secret 的运营配置与映射：登录方式开关、Google One Tap 开关、Creem environment、Creem product ID 映射、发件邮箱、统计 ID、公开功能开关等。

常见配置入口：

- 登录与三方 OAuth：`/admin/settings/auth`
- 支付（Stripe/PayPal/Creem）：`/admin/settings/payment`
- 文件存储（Cloudflare R2）：先通过 runtime 输入配置 R2 bindings 和 `STORAGE_PUBLIC_BASE_URL`，再使用基于存储的上传能力
- 邮件（Resend）：`/admin/settings/email`

更详细的工程文档见：`docs/guides/*.md`。
